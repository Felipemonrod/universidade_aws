⚡ Teste de Elasticidade

Gera carga de CPU para acionar o HPA do Kubernetes e demonstrar o escalonamento automático de pods. A trava de segurança interrompe o teste automaticamente se a CPU ultrapassar o limite configurado.

📊 Status em tempo real
Estado
CPU atual
Threads ativas
Limite CPU
Atualizado a cada 2s automaticamente.
🎛️ Controle do Teste
Cada thread executa Fibonacci recursivo sem parar.
O teste para automaticamente quando a CPU ultrapassar este valor.
⚠️ Atenção: este teste consome CPU real do seu computador. Use um limite conservador (ex: 70–80%) para não travar a máquina.
📋 Como observar o Kubernetes escalonando

Com o teste em execução e o Kubernetes ativo, abra um terminal e execute:

# Observa o HPA em tempo real (atualiza a cada 5s)
kubectl get hpa -n universidade -w

# Observa os pods sendo criados/destruídos
kubectl get pods -n universidade -w

# Ver uso de recursos por pod
kubectl top pods -n universidade

Quando a CPU média dos pods ultrapassar 60% (configurado no HPA), o Kubernetes criará automaticamente novos pods — até o máximo de 10. Quando a carga diminuir, os pods extras serão removidos após ~5 minutos.
